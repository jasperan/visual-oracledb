## What This Is

Interactive showcase website for Oracle AI Database features. Pure client-side static site (no backend, no DB connections) built with Next.js 16.2.1, deployed to GitHub Pages. Each Oracle feature gets its own self-contained widget component with animated visualizations.

## Commands

```bash
npm run dev        # Local dev server (http://localhost:3000)
npm run build      # Production static export to out/
npm run lint       # ESLint (runs `eslint`, not `next lint`)
npm run typecheck  # tsc --noEmit
```

No test framework is configured. Quality checks are `lint` + `typecheck`. There is a `src/app/test/page.tsx` route for manual widget testing during development (not an automated test).

## Architecture

**Stack**: Next.js 16.2.1 (static export) + React 19 + TypeScript 5 + Tailwind CSS v4

**Key deps**: `clsx` + `tailwind-merge` for class composition, `lucide-react` for icons, `@base-ui/react` for accessible primitives, `tw-animate-css` for animation utilities, `class-variance-authority` for variant styles. The `shadcn` package is the shadcn CLI (not a component library).

**Layout**: Single-page app. `src/app/page.tsx` is the main page (`"use client"`) containing a hero section, sticky nav with anchor links, and widget sections revealed on scroll via IntersectionObserver.

**Widgets** (`src/components/widgets/`): Each Oracle feature is an isolated React component with its own local state (useState/useEffect). No shared state management. All data is client-side (hardcoded or computed). Barrel-exported from `index.ts`.

Current widgets (11 total):
1. **JsonDualityWidget** - Relational <-> JSON dual view with edit sync and role slider
2. **CascadeWidget** - Foreign key cascading updates
3. **PropertyGraphWidget** - Interactive node/edge graph
4. **VectorSearchWidget** - 3D similarity search with click-to-query (uses inverse projection)
5. **HnswIndexWidget** - Multi-layer hierarchical index
6. **RagPipelineWidget** - RAG pipeline stages with dynamic values per run
7. **AcidRaceWidget** - ACID vs non-transactional side-by-side
8. **OnnxInDbWidget** - In-database vs traditional ML inference latency
9. **JsonPathWidget** - SQL/JSON path query with live highlighting
10. **DeepSecurityWidget** - Security/access control demo
11. **RacWidget** - Real Application Clusters demo

**Styling**: Tailwind v4 with custom CSS color variables per widget theme defined in `globals.css`. Google Fonts: Inter (body) + JetBrains Mono (code).

## Deployment

GitHub Pages via `.github/workflows/deploy.yml`. Pushes to `main` trigger build + deploy automatically.

`next.config.ts` sets `basePath` and `assetPrefix` to `/visual-oracledb` only when `NODE_ENV === "production"` (empty strings in dev). All asset references must work with this prefix. `output: "export"` produces a fully static `out/` directory. `images.unoptimized: true` is required because Next.js image optimization is incompatible with static export.

## Gotchas

- **Images**: Always use `unoptimized` or plain `<img>` tags. Next.js `<Image>` with optimization will break the static export.
- **basePath in dev**: `basePath` is empty in dev, so links and asset paths work without the `/visual-oracledb` prefix locally. Don't hardcode the prefix in components.
- **No `next lint`**: The lint script runs `eslint` directly (flat config, `eslint.config.mjs`), not `next lint`.
- **`out/` dir**: Generated by `npm run build`. Already in repo but gitignored in CI. Don't commit it manually.

## Adding a New Widget

1. Create `src/components/widgets/YourWidget.tsx` as a `"use client"` component
2. Export it from `src/components/widgets/index.ts`
3. Add a new section in `src/app/page.tsx` with an `id` for the anchor nav
4. Add the corresponding nav link and CSS color variable for the widget theme in `globals.css`
